UIUX Project 3: A/B Testing

Overview

In this project, I attempt to prove that by making certain minimal styling changes to a given website, we can increase the ease and speed for the user to complete a given task.

To do so, we can use A/B testing to see if such styling changes can make differences that are stastically signficant.

Minor Changes to the Website

When I attempted to complete the task on the given website, I was able to succeed, but the one thing that bugged me a bit while doing it was that I was not sure whether the light blue buttons on the right were clickable or not. There was no semblance of clickability to be found (no hover effects, button was light-colored), so I made sure to change the buttons to have a hover state and a darker color.

I also had trouble finding which button belonged to which appointment, so I added an hr element to divide up the buttons (I wanted to put a horizontal line, but I was not able to get it to show up, but the spacing is also better).

I then made two more minor changes: one was to change the color of the physician name, appointment name, and location to be different, and another to order the dates in chronological order. I wanted to group the appointments by physician or location but could not get it to work with the amount of time given to us to modify the website.

Hypothesis pairs

Here, I come up with null and alternative hypotheses for 3 different metrics that I can measure with the website.

Metric 1: Misclick Rate

Null hypothesis:

There is no difference between the misclick rate of version A in comparison to version B.

I predicted that I would be able to reject this null hypothesis (since people are accustomed to where to click for the appointment).

Alternative hypothesis:

The misclick rate of version B is different than the misclick rate of version A.

I made this alternative hypothesis because with the darker colored buttons and the hover effects I added to the buttons (I also grouped the buttons with the appointments), there will probably be less misclicks.

Metric 2: Time on Page

Null hypothesis:

There is no difference between the time spent on the page of version A and the time spent on the page of version B.

I predicted that I would reject this null hypothesis. This is slightly connected to the misclick rate, as you would spend longer on the page if you misclick more times.

Alternative hypothesis:

The time spent on the page of version B is shorter than the time spent on the page of version A.

Given that I changed the buttons as described in the “misclick rate” section, if the user can succeed on the task faster, they will spend less time on the page.

Metric 3: Time to first click

Null hypothesis:

There is no difference between the time to the first click on version A and the time to the first click on version B.

I predicted that I would probably not reject this null hypothesis. If I had made more improvements to the layout to make finding “Adam Ng” and the hospital easier, then perhaps I would reject this null hypothesis (however, I did not have time to do this).

Alternative hypothesis:

The time to the first click on version B is shorter than the time to the first click on version A.

I made this alternative hypothesis because if I make accessibility improvements to the webpage (such as grouping buttons and making the font different colors to distinguish location and physician), it should in theory take a shorter time to find and click a target button.

Statistical Analysis

Here, I run various statistical tests to analyze whether my changes were statistically significant using 3 different metrics.

Metric 1: Misclick Rate

Chosen test:

I chose the chi-squared test because here we are examining the frequency of misclicks (the 2 categories here are misclicked and non-misclicked).

Is the result statistically significant?

Yes!

Numbers that support this:

The chi-squared value is a 12.23, which results in a p-value of a staggering 0.0004, which means that there is a 0.04 percent chance that (given that there is no difference in the change) this happened by chance, which is very low and way below 5 percent.

Conclusion:

We are confident that the alternative hypothesis is true (that there is a difference of misclick rate between version B and version A).

Metric 2: Time on Page

Chosen test:

I chose the one-tailed t-test, as our alternative hypothesis checks to see if the time on page is “shorter” for B than for A, so we would like to see statistical significance for a continuous variable in one direction.

Is the result statistically significant?

Yes!

Numbers that support this:

The t-value is -43.56, meaning that the new average is 43.56 standard deviations away from the average, which results in a p value of less than 0.0001, which is way less than 0.05. A less than 0.01 percent chance of this difference happening by chance shows that the null hypothesis is quite unlikely.

Conclusion:

We are confident that the alternative hypothesis is true (that there is a shorter time spent on the page for version B compared to version A).

Metric 3: Time to first click

Chosen test:

I chose the one-tailed t-test for this. Same reasoning as the above two; the alternative hypothesis requests that the time to the first click is “shorter”, so we are looking for statistical significance in one direction.

Is the result statistically significant?

Yes!

Numbers that support this:

The t-value is -16.04, meaning that the new average is 16.04 standard deviations away from the version A average, which results in a p-value of also less than 0.0001. A less than 0.01 percent chance of this difference happening by chance shows that the null hypothesis is quite unlikely.

Conclusion:

We are confident that the alternative hypothesis is true (that there is a shorter time to the first click for version B compared to version A).

Summary Statistics

Here are some summary statistics regarding the individual websites.

Metric 1: Misclick Rate

This is a chi-squared test, so the counts are the summary statistic (but this is also just the data itself). There were 12 people who succeeded and 12 people who misclicked for version A, and all people (22) succeeded for version B, and the expected value was 6.63 (misclick) and 17.36 (success). Both of these are quite far from 12 and 12 and 0 and 22, so this shows that the change in version B was most likely quite significant in the amount of misclicks and successes.

Metric 2: Time on Page

Version A:

The average time spent on the page is 36139 ms, the median is 36748.5 ms and the variance is 212313710 ms. 24 data points were collected.

This means that the performance of people completing the task varied on the high end (some were lucky and got it on the first try, but many others were not).

Version B:

The average time spent on the page is 8958.5 ms, the median is 8362 ms, and the variance is 8563820.7 ms. 22 data points were collected.

This means that people were consistently able to complete the task quickly on this version of the website.

Metric 3: Time to first click

Version A:

The average time to the first click is 14476.875 ms, the median time is 12491.5 ms, and the variance is 70695020.29 ms. 24 data points were collected.

This means that people's hesistance varied greatly on the high end (some were able to pinpoint down the interface easily and make a decision (whether right or wrong), while others were not).

Version B:

The average time to the first click is 5795.545 ms, the median time is 5553.5 ms, and the variance is 6444395.022 ms. 22 data points were collected.

This means that people were consistently able to determine with confidence what seemed to be the right choice on this website.

Ending Comments

This was a good exercise to attempt to analyze data with A/B testing. However, even if the tweak in the website did make a difference, I think the dramatic effect shown in the statistics was due to the fact that the same population tried the challenge twice and were used to finding where the buttons were (and in that way, the data is very biased). However, we were not equipped with the resources to salvage a large sample of data in this class, so it is understandable.
